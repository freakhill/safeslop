# 0097 — Progressive network session grants

Status: planned
Date: 2026-07-12

SCOPE: implement the first `specs/0089-network-authority-ayo-flo.md` progressive network slice: operator-invoked, session-scoped, exact FQDN:port grants for `environment:"container"` + `network:"deny"` sessions. Grants are runtime overlay state, not profile policy, and update the existing Squid-enforced deny topology without moving the agent onto the egress network.

OFF-LIMITS: do not add `network:"progressive"`, `network:"ask"`, or agent-triggered modal prompts; do not mutate `profile.egress`; do not grant IP literals, private/link-local/metadata, broker/mint endpoints, wildcards, suffixes, TLD globs, CIDRs, raw URLs, or non-HTTP/TCP authority; do not present host/network:allow sessions as enforceably grantable; do not weaken existing base/package/agent/credential/profile allowlists; do not open traffic on observation, UI, overlay write, proxy reload, or control-channel failure.

WORKTREE: `.worktrees/0096-contained-hybrid-default-profiles/`

## Design

Problem: contained-hybrid defaults need "progressive network" while preserving safeslop's current enforceable posture: container deny traffic goes through Squid; host/network:allow are not enforceable; `profile.egress` remains static trusted policy.

Chosen approach: keep `#Network` as `"deny" | "allow"`. Add **session egress grants** as value-free, session-scoped overlay records. Each grant is one exact proxy-observed or operator-entered `FQDN:port`. The runtime materializes grants into a generated Squid include file with one ACL pair per grant, reloads Squid fail-closed, and records grants on the session. Observations are non-modal: a command surfaces denied proxy log entries; the operator chooses whether to grant.

## Contracts

### Grant shape

Add to `internal/engine/session/session.go`:

```go
type EgressGrant struct {
    ID        string    `json:"id"`
    Host      string    `json:"host"`
    Port      int       `json:"port"`
    Source    string    `json:"source"` // "operator"
    CreatedAt time.Time `json:"created_at"`
}
```

`Session` gains:

```go
EgressGrants []EgressGrant `json:"egress_grants,omitempty"`
GrantRevision int          `json:"egress_grant_revision,omitempty"`
```

Grant IDs are stable short IDs derived from normalized `host:port` plus creation order/session ID, or random non-secret IDs. Grants carry no credential values, URL paths, query strings, headers, request bodies, or secret refs.

### Validation

A grant target must be an **exact FQDN:port**:

- `host` lowercased and IDNA-normalized if support is available; otherwise ASCII FQDN only in MVP;
- no leading dot, wildcard, suffix, URL scheme, path, query, fragment, spaces, or slash;
- no IP literal (IPv4/IPv6/bracketed), localhost, metadata host, private/link-local class, broker/mint endpoint, or reverse-DNS inference;
- `port` is `80` or `443` in MVP, matching current Squid safe ports;
- only valid for `environment:"container"` + `network:"deny"` sessions.

Invalid targets return `INVALID_ARGUMENT` and never touch runtime files or session state.

### Effective allow set

For container deny sessions the effective set is still:

```text
base allowlist
∪ policy.AgentEgress(agent)
∪ package RuntimeEgress
∪ policy.CredsEgress(profile)
∪ profile.egress
∪ session egress grants
```

`profile.egress` stays static policy and is never mutated by grant/revoke commands. Session grants are stored on the session record and revoked when the session is removed/stopped/pruned with its runtime dir.

### Squid overlay

Change the proxy materialization from a single file bind to a runtime directory include model so running sessions can update grant overlays safely:

- mount the runtime dir read-only into proxy, e.g. `/safeslop/runtime:ro`;
- `squid.conf` reads:
  - `/safeslop/runtime/allowlist.domains` for static allowlist;
  - `/safeslop/runtime/session-grants.conf` for typed exact grants;
- `session-grants.conf` is generated by safeslop, not user-editable.

Generated grant include shape:

```squid
# generated by safeslop; session <id>; revision <n>
acl grant_0_host dstdomain -n example.com
acl grant_0_port port 443
http_access allow grant_0_host grant_0_port
```

The include is placed after hard denies (ports, metadata/private, IP-literal) and before the final `http_access deny all`. This preserves hard deny ordering and exact host+port pairing.

Overlay update algorithm for a running session:

1. Validate target and session posture.
2. Build the full desired grant list from current session state plus/minus the requested change.
3. Render `session-grants.conf.next` in the runtime dir.
4. Atomically rename to `session-grants.conf` where the backend makes that visible through the runtime directory bind; otherwise write-in-place with a temp+fsync fallback explicitly tested per backend.
5. Run Squid reconfigure in the proxy service (`squid -k reconfigure` or backend equivalent).
6. If reconfigure fails, restore the previous file and reconfigure again; if restore/reconfigure fails, preserve or return to the previous more-restrictive state and report failure. Never attach the agent to the egress network.
7. Save the session record only after the overlay is active.

For created/not-yet-running sessions, grant records may be saved and materialized on first run; launch must fail closed if grant overlay materialization fails.

### Observation channel

First implementation observation channel: parse Squid proxy access logs for denied requests for the session's proxy container.

- Add Squid access logging sufficient to observe denied `CONNECT host:port` and HTTP `Host` requests without URL paths/query strings where possible.
- Add container helper(s) that read proxy logs via the active backend (`compose logs proxy` or backend-equivalent) and parse value-free observations:

```go
type EgressObservation struct {
    Host      string    `json:"host"`
    Port      int       `json:"port"`
    LastSeen  time.Time `json:"last_seen"`
    Count     int       `json:"count"`
    Grantable bool      `json:"grantable"`
    Reason    string    `json:"reason,omitempty"`
}
```

Observation failure returns an empty list plus a warning/error field; it must not open traffic or invent suggestions. Observations are non-modal; only an explicit grant command changes the overlay.

### CLI

Add `session egress` subcommands:

```bash
safeslop session egress observations --session-id ID --output json
safeslop session egress grants --session-id ID --output json
safeslop session egress grant --session-id ID --host example.com --port 443 --output json
safeslop session egress revoke --session-id ID --grant-id G --output json
```

Contracts:

- `observations` lists denied exact FQDN:port observations and grantability reasons.
- `grants` lists active session grants.
- `grant` validates, applies overlay if running, saves session, and returns the updated grant list and `egress_grant_revision`.
- `revoke` removes one grant, applies overlay if running, saves session, and returns updated grants.
- Host sessions and `network:"allow"` sessions return `INVALID_ARGUMENT` or a dedicated non-enforceable contract message: `network grants are only enforceable for container deny sessions`.

`session create/list/status` should include `egress_grants` and `egress_grant_revision` when non-empty. Existing credential scope output remains value-free.

### Emacs/UI

Add non-modal controls only:

- show denied observations in session/status surfaces;
- grant exact selected `FQDN:port` for this session;
- revoke active grant;
- no modal prompt triggered by agent traffic;
- no persistent profile edit in this slice. Copy/suggest persistent CUE edits later only through explicit policy edit/trust.
